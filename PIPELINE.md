# DART AI Real Estate — System Pipeline Architecture

This document outlines the end-to-end technical pipeline of the DART AI platform, detailing how data flows from a raw CSV file to actionable real estate intelligence.

---

## 1. Data Ingestion & Sanitation
**Handler:** `server/services/csv.js`

When a user uploads a CSV or the demo dataset is loaded:
1. **Auto-Detection:** The system detects the delimiter (comma, semicolon, or tab) and file encoding (UTF-8 or Latin-1).
2. **Fuzzy Column Mapping:** A set of 16 core fields (e.g., `price_per_m2`, `area_m2`, `luxury_score`) are mapped using fuzzy string matching to support diverse dataset formats.
3. **Validation & Cleaning:** 
   - Non-numeric prices are discarded.
   - Outliers are filtered (e.g., area < 10m², Price/m² > 500k).
   - Missing coordinates or categorical values are handled gracefully.

## 2. ML Engine: The Random Forest Pipeline
**Handler:** `server/services/model.js`

Once the data is cleaned, the **Random Forest Regression** model is initialized:
1. **Dynamic Feature Engineering:** 
   - **One-Hot Encoding:** Categorical data like City, Property Type, and Compound are converted into binary feature columns.
   - **Cyclical Encoding:** Months are converted into Sine/Cosine waves to help the model understand seasonal real estate trends.
2. **Model Training:** 
   - The dataset is split 80/20 (Train/Test).
   - A Random Forest with 20 trees and a max-depth of 8 is trained.
3. **Performance Metrics:** The server calculates the **R² Score** (accuracy) and **RMSE** (average error) to quantify the model's reliability on that specific dataset.
4. **Feature Importance:** The model ranks which variables (e.g., USD rate vs. Luxury score) have the highest impact on pricing.

## 3. Real-Time Pricing (Prediction)
**Endpoint:** `POST /api/predict` | **UI:** `advisor.js`

When a user submits a "Price Advisor" request:
1. **Scenario Routing:**
   - **Buyer Tab:** Explicitly reads user-defined economic inputs (USD Rate, Material Costs).
   - **Seller Tab:** Auto-injects current market defaults (Default USD Rate, Current Month) to provide a "now" price without user friction.
2. **Inference & Adjustments:** 
   - The base prediction is generated by the Random Forest.
   - **Post-Model Adjustments:** Market-driven coefficients for **Delivery Status** (Off-plan discounts) and **Finishing Quality** are applied to the raw prediction.
3. **Verdict Generation:** The "Fair Price" is compared to the user's "Entered Price" to generate signals like "Undervalued" or "Overpriced".

## 4. Rental Analytics & Yield Engine
**Endpoint:** `POST /api/rental-predict` | **UI:** `rental.js`

This parallel pipeline focuses on financial performance:
1. **Mathematical Financials (The "Numbers"):** 
   - Calculates **Gross Yield**, **Net Yield**, and **Cash-on-Cash** returns.
   - Computes a custom **ROR (Return on Rent) Score** based on payback years and leverage efficiency.
   - Generates an **Annual Cashflow Waterfall** visualization.
2. **AI Market Intelligence (Groq Integration):**
   - **Engine:** Integrated with the **Groq Cloud API** using the `llama-3.3-70b-versatile` model for ultra-low latency inference.
   - **Grounding without Web Search:** Since LLMs have a knowledge cutoff, the system uses **Context Injection** to ground the AI. We feed the model a specific "Geography Reference" block (hardcoded in `rental.js`) containing current rental tiers for major Cairo districts (Zamalek, Maadi, New Cairo, etc.).
   - **Structured Reasoning:** The AI is instructed via a strict JSON schema to analyze the mathematical results from Phase 1 and provide qualitative insights:
     - **Demand Level:** High/Medium/Low based on location and property type.
     - **Tenant Profile:** Who is the likely renter (e.g., "Expat families" vs "Young professionals").
     - **Liquidity Rating:** How fast the unit can be rented out.
   - **Zero-Latency Logic:** By using Groq's LPU (Language Processing Unit), the AI analysis appears almost instantly alongside the mathematical results.

## 5. Frontend & Visualization
**Handlers:** `public/js/app.js`, `public/js/pages/*.js`

The final layer is a modern Single Page Application (SPA):
* **State Management:** Routing and Tab-switching manage visibility without page reloads.
* **Visualization Layer:** `Chart.js` translates backend statistics into interactive Trends, Distribution Bells, and Correlation Heatmaps.
* **Theme Engine:** A sleek Glassmorphism design supports dynamic Light/Dark mode transitions.
